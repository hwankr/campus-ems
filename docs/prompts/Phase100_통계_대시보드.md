# Phase 100 — 통계 대시보드

> 지도 단일 화면을 넘어 건물별 전력 사용량, 태양광 잠재량, 투자 판단을 비교하는 별도 통계 페이지를 만든다. 발표용 요약이 아니라 프로젝트 분석 깊이를 늘리는 기능이다.

---

```
CEMS에 통계 대시보드를 추가한다.

목표:
- 현재 / 단일 지도 화면은 유지한다.
- /statistics 라우트를 새로 만들고 navbar에서 지도와 통계를 오갈 수 있게 한다.
- 통계 페이지는 건물별 비교 분석 화면이다.
- 전력 사용량, 태양광 잠재량, 투자 판단 세 축을 모두 볼 수 있어야 한다.

작업:
1. 공통 navbar 추가
   - components/AppNav.tsx 생성
   - 좌측: CAMPUS-RE100
   - 우측: 지도, 통계 링크
   - 현재 경로를 기준으로 활성 링크 표시
   - app/page.tsx 기존 헤더에 자연스럽게 통합하거나, 지도 전용 컨트롤과 분리해서 상단 구조를 정리
   - app/statistics/page.tsx에서도 같은 navbar 사용

2. 통계 계산 유틸 추가
   - lib/statistics.ts 생성
   - Building[], MonthlyElectricity[]를 받아 건물별 분석 row를 만든다.
   - 최근 12개월 기준으로 다음 값을 계산:
     - annualUsageKwh
     - annualCostKrw
     - annualCo2Kg
     - annualSolarPotentialKwh
     - selfSufficiencyRate
     - annualSavingsKrw
     - installCostKrw
     - paybackYears
     - investmentScore
   - 태양광 계산은 기존 lib/solar-calculations.ts 함수를 재사용한다.
   - 지도에 표시되는 경산캠퍼스 건물 96개와 통계 대상이 맞도록 기존 로더 계약을 재사용한다.

3. 통계 페이지 생성
   - app/statistics/page.tsx 생성
   - 클라이언트 컴포넌트로 구현해 public/data JSON을 로드한다.
   - 기존 loadGyeongsanBuildings(), loadMonthlyElectricity()를 재사용한다.
   - 로딩, 에러, 빈 결과 상태를 모두 표시한다.

4. 탭형 대시보드 UI
   - 탭 3개:
     a. 전력 사용량
     b. 태양광 잠재량
     c. 투자 판단
   - 세 탭은 같은 필터를 공유한다.
   - 탭 변경 시 차트 기준과 테이블 정렬 기준이 바뀐다.
   - lucide 아이콘을 탭 또는 주요 버튼에 사용한다.

5. 필터
   - 구역 필터
   - 건물명 검색
   - 용도 텍스트 검색
   - 상위 10개 / 전체 보기 토글 또는 선택
   - 필터 결과가 없으면 "조건에 맞는 건물이 없습니다." 표시

6. 차트와 테이블
   - Recharts 사용
   - 전력 사용량 탭:
     - 건물별 최근 12개월 사용량 막대 차트
     - 테이블 주요 컬럼: 건물명, 구역, 사용량, 전기료, CO2
   - 태양광 잠재량 탭:
     - 건물별 예상 발전량 막대 차트
     - 테이블 주요 컬럼: 건물명, 구역, 예상 발전량, 자급률, 절감액
   - 투자 판단 탭:
     - 투자 점수 또는 회수기간 중심 차트
     - 테이블 주요 컬럼: 건물명, 구역, 투자 점수, 회수기간, 절감액, 자급률
   - 숫자는 Intl.NumberFormat("ko-KR")로 포맷한다.
   - kg 단위 CO2는 톤 단위로 표시한다.
   - kWh는 큰 숫자가 길어지면 MWh 또는 GWh로 줄인다.

7. 반응형
   - 데스크톱: 좌측 필터 + 우측 분석 영역
   - 모바일: 상단 필터 + 아래 차트/테이블
   - 텍스트가 버튼과 테이블 셀에서 겹치지 않게 줄바꿈/축약 처리

8. 테스트
   - lib/statistics.ts 단위 테스트 추가
   - 최근 12개월 집계, 회수기간 null 처리, 투자 점수 정렬 기준을 검증한다.

검증:
- / 에서 navbar의 통계를 클릭하면 /statistics로 이동한다.
- /statistics에서 지도를 클릭하면 /로 돌아간다.
- 세 탭이 모두 전환되고 차트와 테이블 기준이 바뀐다.
- 구역/건물명/용도 필터가 차트와 테이블에 함께 반영된다.
- 모바일 폭에서 필터, 탭, 테이블이 겹치지 않는다.
- npm test 통과
- npm run lint 통과
- npm run build 통과
```

## 구현 메모

- `investmentScore`는 최근 12개월 사용량과 연간 태양광 잠재량을 각각 최대값 기준으로 정규화한 뒤 조화평균으로 계산한다. 한쪽 값만 큰 건물보다 사용량과 잠재량이 모두 큰 건물이 상단에 오도록 하기 위한 기준이다.
- 화면에서는 투자 점수를 설명하기 애매하지 않도록 "우선 검토 순위"로 표시한다.
- 상위 10개 / 전체 보기 선택은 차트와 테이블에 함께 적용한다. 전체 보기는 가로 스크롤 차트를 허용한다.
- 개발 중 `next dev`가 stale `.next` 산출물 때문에 멈추거나 클라이언트 청크가 404가 되면 dev 서버를 종료하고 `.next`를 재생성한 뒤 다시 확인한다.
